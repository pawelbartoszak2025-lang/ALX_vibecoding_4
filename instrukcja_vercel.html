Jak wrzucić stronę na Vercel

Prosto, krok po kroku — bez wiedzy technicznej. Na końcu Twoja strona z ofertami będzie dostępna pod adresem w internecie, z którego skorzystasz na każdym komputerze i telefonie.

00Czym właściwie jest Vercel?

Vercel to firma, która za darmo udostępnia w internecie takie strony i aplikacje jak Twoja. Wyobraź sobie, że Twój projekt leży teraz tylko na Twoim komputerze — Vercel bierze jego kopię i stawia ją na swoim serwerze, dostępnym dla całego świata pod adresem typu twoja-nazwa.vercel.app.

Nie musisz nic instalować ani znać się na serwerach. Cała sztuczka polega na połączeniu trzech rzeczy:

  • Twój projekt (folder na komputerze) — już go masz.
  • GitHub — taki internetowy „dysk" na kod, z którego Vercel pobiera projekt.
  • Vercel — bierze projekt z GitHuba i publikuje w sieci.

Kolejność jest zawsze ta sama: komputer → GitHub → Vercel.

01Zanim zaczniesz — co przygotować

Załóż dwa darmowe konta (jeśli jeszcze nie masz):

  • Konto na github.com
  • Konto na vercel.com — przy zakładaniu wybierz opcję „Continue with GitHub", to połączy oba konta od razu.

Dobra wiadomość: Twój projekt jest już przygotowany pod Vercel. Masz gotowe pliki vercel.json, requirements.txt oraz api/index.py — to one mówią Vercelowi, jak uruchomić stronę. Nie musisz ich ruszać.

02Krok 1 — wyślij projekt na GitHub

Vercel nie czyta wprost z Twojego komputera — najpierw projekt musi trafić na GitHub. Masz dwie drogi. Wybierz łatwiejszą A, chyba że wolisz pisać komendy.

Droga A — bez komend (GitHub Desktop)

  1. Pobierz GitHub Desktop

    Wejdź na desktop.github.com, zainstaluj i zaloguj się swoim kontem GitHub.

  2. Dodaj swój projekt

    W programie: File → Add Local Repository i wskaż folder ALX_vibecoding_4. Program sam wykryje, że to projekt Git.

  3. Opublikuj

    Kliknij niebieski przycisk „Publish repository". Zostaw zaznaczone „Keep this code private" (kod prywatny — tak bezpieczniej). Zatwierdź.

Droga B — przez komendy (PowerShell)

Jeśli wolisz wpisać polecenia, otwórz PowerShell w folderze projektu i wykonaj kolejno (najpierw załóż puste repozytorium na github.com/new i skopiuj jego adres):

# sprawdź, czy wszystko jest zapisane w Git
git status
git add .
git commit -m "publikacja na Vercel"

# podłącz swoje repozytorium z GitHub (wklej swój adres)
git remote add origin https://github.com/TWOJ-LOGIN/otodom-oferty.git
git branch -M master
git push -u origin master

Jeśli git remote add origin zgłosi, że „origin" już istnieje — pomiń tę linię i od razu zrób git push.

Co zostaje na komputerze: pliki otodom.db (baza) oraz discord_config.json są celowo poza repozytorium (są w .gitignore) — i dobrze, bo zawierają dane i sekrety. Nie wysyłaj ich ręcznie.

03Krok 2 — zaimportuj projekt do Vercela

  1. Wejdź do panelu Vercel

    Zaloguj się na vercel.com/new.

  2. Wybierz swoje repozytorium

    Zobaczysz listę projektów z GitHuba. Znajdź ten z ofertami i kliknij „Import". Jeśli go nie widać — kliknij „Adjust GitHub App Permissions" i daj Vercelowi dostęp do tego repozytorium.

  3. Nie zmieniaj ustawień budowania

    Vercel sam rozpozna projekt Python dzięki plikom, które już masz. Pola Framework Preset Build Command zostaw tak, jak są — domyślnie.

Zanim klikniesz finalny „Deploy", zrób jeszcze krok 3 poniżej (zmienne) — to ważne.

04Krok 3 — dodaj hasła i token (zmienne środowiskowe)

Twój projekt korzysta z tokenu bota Discord, którego nie ma w kodzie (i bardzo dobrze). Trzeba go podać Vercelowi osobno, w bezpiecznym miejscu.

Na ekranie importu rozwiń sekcję „Environment Variables" i dodaj:

  • Name (nazwa): Discord_bot_token
  • Value (wartość): Twój token bota z Discorda

Jeśli pominąłeś ten krok przy imporcie — dodasz to później w Settings → Environment Variables w panelu projektu.

Nigdy nie wklejaj tokenu do kodu ani nie wysyłaj go na GitHub. Zmienne środowiskowe są właśnie po to, żeby trzymać sekrety osobno.

04bKrok 3b — trwałe dane (Supabase) opcjonalne

Domyślnie Vercel nie zapisuje danych trwale między uruchomieniami (serwer ulotny). Jeśli chcesz, żeby oferty, kryteria, harmonogram i pamięć Discord przetrwały redeployment, podłącz darmową bazę danych Supabase. To jednorazowa konfiguracja.

  1. Utwórz schemat bazy w Supabase

    Zaloguj się na supabase.com, wejdź do swojego projektu, a następnie:
    SQL Editor → New query — wklej całą zawartość pliku supabase_schema.sql (znajdziesz go w folderze projektu) i kliknij „Run". Wystarczy zrobić to raz — zostaną utworzone tabele oferty, app_settings i discord_sent.

  2. Skopiuj adres projektu i klucz API

    W panelu Supabase przejdź do Project Settings → API. Znajdziesz tam dwie wartości, które zaraz wkleisz do Vercela:

    • Project URL — adres wyglądający jak https://xxxx.supabase.co
    • service_role (sekcja „Project API keys") — długi klucz zaczynający się od eyJ…

  3. Dodaj zmienne środowiskowe na Vercelu

    W panelu Vercela wejdź do Project Settings → Environment Variables i dodaj dwie pozycje:

    • Name: SUPABASE_URL  ·  Value: skopiowany Project URL
    • Name: SUPABASE_KEY  ·  Value: skopiowany klucz service_role

  4. Zrób Redeploy

    Wejdź w zakładkę Deployments, kliknij „···" przy ostatnim wdrożeniu i wybierz „Redeploy". Od tej chwili oferty, kryteria, ustawienia harmonogramu i lista wysłanych wiadomości Discord są trwale przechowywane w Supabase — przetrwają każde kolejne wdrożenie.

Ważne — klucz service_role to sekret. Daje pełny dostęp do bazy z pominięciem zasad bezpieczeństwa. Trzymaj go wyłącznie w zmiennych środowiskowych Vercela — nigdy nie wklejaj go do kodu ani nie wysyłaj na GitHub.

04cImport kursów walut NBP (kursy_walut)

Jeśli chcesz mieć w bazie Supabase historyczne kursy walut pobrane z Narodowego Banku Polskiego, uruchom jednorazowy skrypt nbp_import.py. Poniżej krok po kroku.

  1. Utwórz tabelę w Supabase

    W panelu Supabase otwórz SQL Editor → New query, wklej całość pliku supabase_schema.sql i kliknij Run. Powstanie m.in. tabela kursy_walut (jeśli już raz uruchamiałeś ten plik, nic złego się nie stanie — tabele tworzone są tylko, gdy ich nie ma).

  2. Ustaw zmienne środowiskowe

    Na komputerze ustaw zmienne SUPABASE_URL i SUPABASE_KEY (te same, co dla aplikacji).

  3. Uruchom skrypt

    Uruchom: python nbp_import.py. Skrypt pobierze kursy wszystkich walut (tabele A i B) za ostatnie 18 miesięcy i zapisze je do tabeli kursy_walut, pokazując postęp.

  4. Powtarzanie jest bezpieczne

    Skrypt można uruchomić ponownie — istniejące kursy zostaną nadpisane, bez duplikatów.

05Krok 4 — opublikuj finał

  1. Kliknij „Deploy"

    Vercel zacznie budować stronę. Potrwa to zwykle kilkadziesiąt sekund.

  2. Poczekaj na „Congratulations"

    Gdy zobaczysz gratulacje i zrzut ekranu Twojej strony — gotowe.

  3. Otwórz swój adres

    Kliknij „Visit" albo wejdź na twoja-nazwa.vercel.app. Strona działa w internecie!

06⚠ Ważne — co zadziała, a co nie

Muszę być z Tobą szczery: ten projekt powstał jako aplikacja lokalna (działająca na Twoim komputerze). Vercel świetnie nadaje się do pokazania strony, ale ma inny sposób działania, więc niektóre funkcje nie zadziałają tak samo. Po ludzku:

✅ Zadziała

  • Wyświetlanie samej strony i jej wyglądu (lista ofert, panele, układ).
  • Strona będzie dostępna z każdego urządzenia, pod stałym adresem.

⚠ Może nie działać tak, jak na komputerze

  • Zapamiętywanie danych (baza otodom.db). Vercel po każdym uruchomieniu „zapomina" zapisane pliki — to tzw. serwer ulotny. Oferty, konta i ustawienia nie zostaną trwale zapisane. Lokalnie działa, bo plik bazy leży na Twoim dysku.
  • Harmonogram (automatyczne pobieranie co 10–15 min). Na Vercelu nie ma „programu w tle", który chodzi cały czas — strona budzi się tylko, gdy ktoś ją odwiedzi. Automat się nie uruchomi.
  • Bot Discord. Bot musi działać non-stop, a Vercel tego nie zapewnia. Bota najlepiej zostawić uruchamianego z Twojego komputera (przez Oferty.vbs).
Wniosek prostym językiem: Vercel idealnie nada się do pokazania komuś, jak strona wygląda i działa „na żywo". Jeśli natomiast potrzebujesz trwałej bazy i automatycznego pobierania ofert 24/7 — albo zostaw to na swoim komputerze, albo przejdź na hosting z prawdziwym, ciągle działającym serwerem (np. Railway, Render, mały VPS). Chętnie pomogę wybrać i ustawić, jeśli zechcesz iść tą drogą.

07Jak później wprowadzać zmiany?

Najlepsze w tym układzie: nie musisz nic robić ręcznie na Vercelu. Wystarczy, że wyślesz zmiany na GitHub, a Vercel sam zauważy i zaktualizuje stronę.

git add .
git commit -m "opis zmiany"
git push

Albo w GitHub Desktop: wpisz krótki opis na dole, kliknij „Commit", potem „Push origin". Po chwili nowa wersja jest w internecie.
